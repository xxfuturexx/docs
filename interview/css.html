# 面试题：介绍一下CSS的盒子模型

css的盒子模型有哪些： 标准盒子模型（W3C标准盒模型）、IE盒子模型（怪异盒模型）
css的盒子模型的区别：
	标准盒子模型：margin、border、padding、content
	IE盒子模型：margin、content(border + padding + content)
通过css如何转换盒子模型：
 box-sizing: content-box;/*默认是标准盒子模型*/ 元素的宽高 = 边框 + 内边距 + 内容高度
 box-sizing: border-box;/*IE盒子模型*/		  元素的宽高 = widtn/height的宽高（为了保证盒子元素的宽高不变，会自动减去一部分内容的宽高）

<!-- CSS的盒子模型.html -->
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
    <style>
        *{
            margin: 0;
            padding: 0;
        }
        ul{
            list-style: none;
            overflow: hidden;
        }
        .ul1 li{
            box-sizing: content-box;/*默认是标准盒子模型*/
            margin: 10px;
            padding: 50px;
            width: 200px;
            height: 200px;
            border: 5px solid #cccccc;
            float: left; /*盒子高度是 200 + 50 + 50 + 5 + 5 = 310  元素的宽高 = 内容高度 + 上下边框 + 上下内边距 */
        }
        .ul2 li{
            box-sizing: border-box;/*IE盒子模型*/
            margin: 10px;
            padding: 50px;
            width: 200px;
            height: 200px;
            border: 5px solid #cccccc;
            float: left; /*盒子的高度是 200 元素的宽高 = widtn或者height的高度*/
        }


    </style>
</head>
<body>
    <ul class="ul1">
        <li>1</li>
        <li>2</li>
        <li>3</li>
    </ul>
    <hr>
    <ul class="ul2">
        <li>1</li>
        <li>2</li>
        <li>3</li>
    </ul>
</body>
</html>

标准盒子模型(content = css设置的width ） 但是元素盒子的宽高是 = 边框 + 内边距 + 内容高度

IE盒子模型(content 的width【width是css设置的width200】包含border和padding,所以 元素盒子的宽高 = widtn/height的宽高）

# 面试题：flex布局

https://www.cnblogs.com/qqfontofweb/p/11225696.html

https://www.cnblogs.com/10yearsmanong/p/12212825.html

//语法
.box{
  display: flex;
}

Flex:

Flex是Flexible Box的缩写，意为”弹性布局”，用来为盒状模型提供最大的灵活性。

Flex布局有两层,采用flex布局的元素称为flex容器,其子元素则自动成flex item,即项目.

Flex布局（flex容器）:

  • justify-content:定义了项目在主轴上的对齐方式。(justify)
  • align-items:定义项目在交叉轴（侧轴）上如何对齐。

Flex item属性（给子元素设置的属性）:

order属性定义项目的排列顺序。数值越小，排列越靠前，默认为0。

flex-grow属性定义项目的放大比例，默认为0，即如果存在剩余空间，也不放大。

# 面试题：对BFC规范(块级格式化上下文：block formatting context)的理解？

BFC就是页面上一个隔离的独立容器，容器里面的子元素不会影响到外面的元素

1.了解BFC:块级格式化上下文
2.BFC的原则：如果一个元素具有BFC，那么内部元素再怎么弄，都不会影响到外面的元素
3.如何触发BFC：
	float的值非none，left或者right
	overflow的值非visible,overflow: hidden;overflow: scroll;
	display的值为：inline-block、table-cell...
	position的值为:absolute、fixed

还未触发BFC前( ul的边框会乱飘 )

触发BFC后：

    <style>
        *{
            margin:0;
            padding:0;
        }
        ul{
            list-style: none;
            border: 5px solid #cccccc;
            overflow: hidden;/*给ul加这一句触发BFC，  display: inline-block;position:absolute;也可以 */
          
        }
         ul li{
             width: 200px;
             height: 200px;
             margin: 10px;
             background-color: skyblue;
             border: 5px solid #cccccc;
             float: left;/*不是li乱飘是ul所以给ul加，然后ul触发BFC*/
         }
    </style>


<body>
    <ul>
        <li>1</li>
        <li>2</li>
        <li>3</li>
    </ul>
    <h1>我是h1</h1>
</body>

触发了BFC的规范或者特性，就会让其中的子元素做任何改变都不会影响上面的元素和下面的元素

overflow: hidden;overflow: scroll;
float:不是none

# 面试题：position有哪些值?有什么作用? 【特别多公司问】

static（静止的）[默认值] 没有定位（当框架有position:absolute,可以使用position:static没有定位来覆盖）
fixed  		固定定位，相对于浏览器窗口进行定位
relative	相对于自身定位，不脱离文档流
absolute	相对于第一个有relative的父元素，脱离文档流


relative和absolute区别:
1.relative不脱离文档流、absolute脱离文档流
2.relative相对于自身、absolute相对于第一个有relative的父元素（子绝父相）
3.relative当有left、right、top、bottom属性时 ==>只显示left、top
	absolute有left、right、top、bottom属性时 ==> 会正常显示left、right、top、bottom

# 面试题：display: none;与visibility: hidden;的区别

重绘与回流(重排)的概念

回流(重排): 当浏览器必须重新处理和绘制部分或全部页面时，回流就会发生(回流去前面一步然后重绘)

重绘: 不影响布局, 只是标签页面发生变化, 重新绘制

注意: 回流(重排)必引发重绘, 重绘不一定引发回流(重排)

浏览器在输入url后会执行什么步骤

display:none;与visibility:hidden
功能：都是隐藏元素

区别：
1.display:none不占位置，visibility:hidden占位置
	原理：display:none			第一次不绘制,第二次才绘制
	      visibility:hidden	  第一次会绘制，所以占位置
          
2.		visibility:hidden重绘        display:none 重绘和回流	
触发回流必定会触发重绘
触发重绘不一定会触发回流

重绘：盒子的font-size、background...
回流：尺寸大小、位置(left、top)

# 面试题：DOCTYPE标签 doctype（声明、文档类型）

定义和用法:
<!DOCTYPE> 声明必须是 HTML 文档的第一行，位于 <html> 标签之前。

<!DOCTYPE> 声明不是 HTML 标签；它是指示 web 浏览器关于页面使用哪个 HTML 版本进行编写的指令。

在 HTML 4.01 中，<!DOCTYPE> 声明引用 DTD，因为 HTML 4.01 基于 SGML。DTD 规定了标记语言的规则，这样浏览器才能正确地呈现内容。

HTML5 不基于 SGML，所以不需要引用 DTD。

二. meta标签如何使用

在head标签中可以添加多个meta标签，meta标签有四大属性：

  • charset属性： 用于声明页面的字符集编码，一定要写在第一行，否则将会出现乱码

    <meta charset="utf-8">

  • http-equiv属性： 定义这个属性会添加到http头部，可用与刷新重定向和设置Cookie等操作，与content属性结合使用

    //不想用JS来进行自动刷新重定向，可以直接用meta标签实现
//令http-equiv为refresh, 在content中可以定义刷新时间和URL
<meta http-equiv="Refresh" content="5;url=http://blog.yangchen123h.cn" />
<meta http-equiv="Set-Cookie" content="cookievalue=xxx; expires=Friday, 12-Jan-2001 18:18:18 GMT； path=/">

  • name属性：常用于设置网站关键字和描述方便SEO，也常用于通过viewport（视口）调整页面的展示方式，要与content属性结合使用

    <!-- 设置网站关键字，方便SEO -->
<meta name="keywords" content="供应德州土工膜、德州土工膜供应，德州土工膜批发市场，德州土工膜批发，德州土工膜行情，德州土工膜">
<!-- 是否启用 WebApp 全屏模式 -->
<meta   name="viewport"  content="viewport-fit=cover, width=device-width, initial-scale=1.0, minimum-scale=1.0, maximum-scale=1.0, user-scalable=no"/>
